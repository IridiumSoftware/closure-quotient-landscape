## Abstract

> Reproduced verbatim from `closure_quotient_landscape.tex`'s
> `\begin{abstract} ... \end{abstract}` block (lines 81–110 of the
> source).

We characterise the closure-quotient family of the
Closure-Forces-Structure (CFS) programme — the gauge-equivalence
quotients $Q_n$ of multiway graphs generated by ternary hypergraph
rewriting — using exact Gaussian-integer ray arithmetic and a
validity-gated probe corpus. Our main result is a *generative algebra*
on the family: for closure quotients $A$ and $B$, the count
$Q(A \circ B) = Q(A) + Q(B) + \delta(\text{interface})$ with $\delta$
a structure-independent interface correction ($\delta_\sqcup = 0$
disjoint, $\delta_{\text{bridge}} = +3$ per bridge triple,
$\delta_{\text{glue-}g} = -(4g-3)$ for a $g$-vertex glue); the counts
form a commutative monoid under disjoint sum $\sqcup$, and the
charge-conjugation doubling rule $|Q \cup C(Q)| = 2|Q|$ is the special
case where $C(Q)$ behaves as a disjoint generic copy of $Q$. We
exhibit the closure laws $\text{cyc}(n) \to 4n$ and
$K_n^3 \to n(3n-1)/2$ (pentagonal), verify the latter exactly for
$n = 6 \ldots 20$ across two independent strict-generic IC pools, and
exhibit a prime closure object $Q_{181}$ via an irregular $12$-vertex
$132$-edge seed. Methodologically, exact arithmetic and validity
gating sharpen the published family documentation: the
previously-listed $Q_{98}$ is corrected to $Q_{102}$ (a Float64
fidelity-clustering artifact), $Q_{180}$ is shown not to exist
($C$-closure is idempotent on $C$-closed objects), and the canonical
s58 initial-condition (IC) pool used throughout the prior CFS
literature is found to pass the $K_6^3 \to 51$ validity gate but
exhibit a specific non-genericity at $K_n^3$ for $n \geq 9$ — so
IC-genericity is a hierarchy (fully-generic / small-$n$-generic /
degenerate), not the binary the literature treats it as.

## Probe reproducibility

```sh
cd paper/scripts/
for s in *.py; do python3 "$s" || echo "FAIL: $s"; done
```

All probes run on CPython 3.10+ with no external dependencies. Total
runtime < 5 minutes on a single CPU core. Every probe runs its
validity gate (`K_6^3 → 51`) before reporting measurements.
